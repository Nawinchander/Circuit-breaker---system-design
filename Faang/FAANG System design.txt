FAANG System Design Use Cases
1. Payment Gateway Calls

If payment provider is failing:

Open breaker
Return graceful fallback
Avoid retry storm

Example:

Order Service → Stripe
2. Recommendation Service

If recommendation engine is down:

Return fallback:

"Top products"

instead of failing the homepage.

3. Email Service

If email provider fails:

Open breaker and queue email for retry later.

4. Database Protection

If DB latency spikes:

Open breaker for expensive queries.



Benefits
1. Prevents Cascading Failure

One failed service won’t bring down all services.

2. Saves Resources

No wasted threads/connections on failing services.

3. Fast Failure

Users get immediate response instead of long timeout.

4. Graceful Degradation

Fallback responses can be returned.

Example:

"Recommendations unavailable"
5. Improves Stability

Helps the system recover.


Drawbacks
1. Added Complexity

More logic to maintain.

2. Wrong Threshold Tuning

If threshold is too low:

Circuit opens unnecessarily

Too high:

System overloads before opening

3. Temporary Denial

Service may recover but breaker is still open until timeout.

State Management in Distributed Systems

If multiple instances exist, breaker state must be synchronized.
