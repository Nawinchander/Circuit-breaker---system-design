Circuit Breaker for Redis Cache Service
Scenario:

Your Product Service fetches product data from Redis.

If Redis is down:

avoid repeated retries
fetch from DB fallback
protect app threads

This is common in:

Amazon product cache
Netflix metadata cache


Product API → Redis Cache
          ↘ if fail → DB fallback


Interview Point

This prevents:

Redis retry storm
Increased latency
DB overload caused by bad retries
