FAANG Interview Insight

These examples show three important circuit breaker strategies:

1. Fallback to another data source

Redis → DB

Primary fails → Secondary works

2. Fallback to alternate provider

SMS → Email

Provider A fails → Provider B works


3. Graceful degraded response

Inventory → "Unknown"

Service fails → Safe response


FAANG Interview Question
Q: Where should we use circuit breakers?

Use them in:

external APIs
payment services
cache services
microservices
notifications
inventory systems

Because they prevent:

cascading failure
thread exhaustion
latency spikes


Q: What fallback strategies exist?
Cached data
Secondary provider
Default response
Queue for later processing
FAANG-Level Interview Answer

“In production systems, circuit breakers are applied around unreliable downstream dependencies like cache, notification, and inventory services. They prevent repeated failures from propagating through the system by opening after threshold failures and returning fallback responses like cached data, alternate providers, or degraded defaults.”

That is exactly the kind of answer interviewers want.
